# Agentic Tool-Use Architecture

This document describes the "On-Demand Retrieval" architecture used by the Chromium Multi-Agent Reviewer. The system has transitioned from a "context-stuffing" model to a high-performance agentic loop that navigates the 30GB+ Chromium codebase surgically using tools.

## 1. The Core Philosophy

Standard LLM workflows attempt to cram all relevant files into the context window at once. In Chromium, a single change to a media component can touch files that are 6,000+ lines long, quickly exceeding context limits and distracting the model with irrelevant code.

Our architecture uses **Recursive Tool Calls**:
1.  **Initial Context**: Agents receive only the `diff.patch`, `commit_info`, and a `project_tree` (directory map).
2.  **On-Demand Discovery**: Agents use `search_files` to find where symbols are defined or called.
3.  **Surgical Retrieval**: Agents use `get_function` or `read_lines` to pull in specific logic only when needed.
4.  **Synthesis**: Findings are summarized into the final review.

## 2. Toolset Reference

### `get_function(file_path, function_name)`
The primary tool for logic analysis. It utilizes a **Hybrid Extraction Strategy**:
-   **Local Step**: Uses a Python-based regex and brace-matching scanner. It is language-agnostic (supporting C++, Java, and Objective-C++) and instantaneous.
-   **LLM Fallback**: If local extraction fails (due to complex macros or unusual formatting), it triggers a sub-task using `gemini-3-flash-preview`.
-   **Windowing Optimization**: The LLM fallback does not ingest the whole file. It calculates a 100,000-character window around the target symbol, ensuring extraction takes seconds rather than minutes.

### `search_files(file_paths[], pattern, fixed_strings, context_lines)`
The "flashlight" tool. It allows agents to grep across multiple files at once.
-   **Explicit Targeting**: Agents must provide a list of paths, reducing ambiguity.
-   **Contextual Awareness**: Supports `context_lines` (e.g., return 2 lines before/after matches) so agents can understand flow without a follow-up call.
-   **Safety**: Automatically truncates output if too many matches are found, forcing the agent to refine its query.

### `read_lines(file_path, start_line, end_line)`
The surgical tool. Used when the agent knows exactly where an issue is (from a grep or stack trace) and needs to see the surrounding logic.

## 3. Performance & Reliability Features

### Context Caching
The tool uses the Gemini **Context Caching API**. The initial core context (diff, summary, project tree) and the tool definitions are baked into a shared cache. All agents (Logic, Memory Safety, etc.) run in parallel against this single cache, drastically reducing "time-to-first-token" and API costs.

### Thread-Safe Memoization
To prevent redundant work across the 10+ concurrent agents:
-   **Disk Cache**: Once a file is fetched from Gerrit, it is saved to the local temporary directory. All subsequent `get_file` or `search_files` calls read from disk.
-   **Extraction Cache**: If the `momory_safety` agent extracts `SetCcLayer`, and the `logic` agent later asks for the same function, the result is returned instantly from an in-memory cache.

### Exponential Backoff with Jitter
To handle the massive request volume generated by 10 agents performing deep recursive traces, the `GeminiClient` implements:
-   **Retriable Codes**: Automatically retries on 429 (Rate Limit), 500, 502, 503, and 504 errors.
-   **Jittered Backoff**: Adds random noise to retry delays to prevent "thundering herd" collisions when multiple agents are throttled.
-   **High Stamina**: Supports up to 100 tool iterations per agent.

### Detailed Audit Log
Every tool call, Gerrit fetch, and LLM turn is logged to `agent_activity.log` in the PR directory. This provides a transparent "trace" of the agent's thought process without cluttering the CLI dashboard.

## 4. Multi-Language Support
The system is optimized for Chromium's polyglot nature:
-   **C++**: Understands `::` namespaces and constructor initializer lists.
-   **Java**: Handles `.` separators and annotations.
-   **Objective-C++ (.mm)**: Brace-matching logic is robust against square-bracket messaging syntax.
